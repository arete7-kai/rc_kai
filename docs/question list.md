## 四、需求要点（答题时的约束背景，非提问）

- 场景：多个内部业务系统在关键事件发生时，需调用**不同外部供应商**的 HTTPS API 进行通知
  （例：注册后通知广告系统 / 付款后通知 CRM / 购买后通知库存系统）
- 不同供应商 API：**请求地址不同、Header / Body 格式不同**
- 业务系统：**不关心外部 API 返回值**，只需保证通知请求被**稳定、可靠地送达**
- 目标：设计并实现一个内部服务，接收业务系统提交的外部 HTTP 通知请求，尽可能可靠地投递到目标地址
